Kabuki-Cord Field Manual

Help & setup

Start safely, shape a character, and collect useful diagnostics without uploading anything.

01

Safe first run

Complete these in order. Green items are already ready.

02

Sign into Discord first

Kabuki-Cord uses a private local browser profile, not the official bot gateway.

  1. Open API & Runtime. Discord email/password are optional conveniences and are stored in the OS keyring.
  2. Click Sign In. Complete login, CAPTCHA, passkey, or 2FA yourself in the visible Discord window.
  3. Keep that window open until the app confirms the session. Then click Sync Discord.
  4. Use Switch Account for another login. It clears only Kabuki-Cord's private Discord session, not your normal browser.
Personal-account browser automation is less stable and carries more platform risk than a supported integration. Start in Observe only.
03

Build a precise character

A custom card is required for persona-driven drafts; observation tools still work without one.

NameThe identity shown in previews and approvals.
System promptCore worldview, voice, limits, and non-negotiable behavior.
AliasesNames and spellings that count as mentions of the character.
Trigger keywordsTopic cues that make a reply more relevant; they are not commands.
Style rulesSentence length, tone, vocabulary, formatting, and recurring quirks.
Engagement rulesWhen to enter, when to stay quiet, and conversations to avoid.
Growth notesPersistent story and behavior details added after the base card.
User guidanceExtra instructions scoped to one user, server, or channel.
Put stable identity in the system prompt, writing mechanics in Style, and routing decisions in Engagement. Test with Review every draft before autonomy.
04

Channel actions

Observation and maintenance do not require a custom character card.

ObserveReads new visible messages into local memory. It does not send.
LatestRereads the newest visible messages in the selected channel.
BackfillScrolls older accessible history into local memory.
ReactAdds rate-limited lightweight emoji reactions when channel rules allow.
EngageAllows reply decisions and drafts. A useful custom card and API setup are needed.
AutoChannel-level permission for unreviewed sends; global mode must also allow it.
Dojo SweepPauses other servers, deep-reviews accessible history, and queues items for manual review.
Repair / SyncRefreshes server and channel discovery when Discord's UI changes or lists are stale.
05

Choose the right response mode

The global mode combines with each channel's Observe, Engage, and Auto switches.

Observe onlyScan and reason, never send. Optional Draft while observing can spend API budget on previews.Best for setup, discovery, and regression testing.
Review every draftEligible messages become approvals. Nothing is sent until you review and click Send.Best default for tuning a new card.
Limited autonomousRegular eligible replies may auto-send; proactive and manual drafts still require review.Best after consistent review results.
Autonomous liveEligible replies can auto-send when Engage and Auto are enabled and budgets and guards pass.Highest risk; watch pacing and the event feed.
06

Collect local diagnostics

Create a support bundle you can inspect and share manually.

Privacy-conscious support bundleIncludes redacted rotating application logs, content-free activity metrics, app version, platform details, and configuration counts. It excludes chat text, character cards, credentials, API keys, and the Discord browser profile.
Nothing is uploaded or sent. The folder opens locally so you can review and copy the ZIP yourself.
