# Refresh Local State V2 Sprite Prompt

CLI fallback image generation used `gpt-image-2` at high quality. The original Refresh sheet was supplied as an action reference only, while the accepted Latest V2 sheet controlled the actor identity, mask, costume, proportions, rendering, palette, camera, lighting, and prop quality.

The final brief requested exactly eight cells in a 4x2 atlas. It locked one state console on the left and one actor on the right throughout this action story: idle with three dim bars and lever up; pull the lever; fill the top bar; fill the top and middle bars; fill all three; press one commit button with a compact core pulse; confirm all bars and the status core healthy; recover to matching idle with the lever returned.

The prompt required exactly one console, one lever, three stacked state bars, one commit button, and one violet/cyan status core. It prohibited console redesigns, changing console height, moving controls, changing the bar count, extra controls, mask portraits, portals, fans, scrolls, stamps, floating panels, large energy vortices, text, logos, watermarks, and any variation in the flat `#00ff00` chroma-key background.

The CLI source is `output/imagegen/refresh-v2-atlas-chromakey.png`, copied into the project as `design_assets/refresh-v2-atlas-chromakey.png`. Automatic border sampling measured the generated key as `#0af610`; the alpha result is `design_assets/refresh-v2-atlas-transparent.png`.
