# Dry Mode V2 Sprite Prompt

CLI fallback image generation used `gpt-image-2` at high quality. The accepted default scanner V2 atlas controlled the Kabuki actor identity, mask markings, costume, proportions, rendering, palette, camera, and scale. The original Dry Mode sheet was supplied only as an action-story reference.

The final brief requested exactly eight cells in a 4x2 atlas. It locked one actor to one ground line throughout a monotonic dehydration story: guarded intact stance; robe edges stiffen and crack; small flakes lift; the same outer edge begins dissolving; the body narrows toward the center; one connected remnant feeds a single vertical dust stream; the stream collapses into one centered mound with a recognizable mask fragment; three final motes fade above the stable residue.

The prompt prohibited fan and weapon appearances, orange smoke pops, explosions, magical auras, multiple piles, duplicate limbs or faces, random glyphs, pose teleporting, text, logos, watermarks, and any variation in the flat `#00ff00` chroma-key background.

The CLI source is `output/imagegen/dry-mode-v2-atlas-chromakey.png`, copied into the project as `design_assets/dry-mode-v2-atlas-chromakey.png`. The alpha-extracted source is `design_assets/dry-mode-v2-atlas-transparent.png`.
