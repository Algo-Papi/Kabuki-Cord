# Full-auto Mode V2 Sprite Prompt

CLI fallback image generation used `gpt-image-2` at high quality. The accepted default scanner V2 atlas controlled the Kabuki actor identity, mask markings, costume, proportions, rendering, palette, camera, and scale. The original Full-auto sheet supplied the exaggerated impossible-pocket action story, and its clearest fully visible frame supplied the theatrical machine-prop design language.

The final brief requested exactly eight cells in a 4x2 atlas. It locked one oversized dark-gunmetal prop with one wooden stock, one rectangular bronze box, one receiver, one top charging handle, one long perforated barrel shroud, one front sight, one muzzle, and one folded bipod throughout this sequence: brace; open one robe fold; draw the same prop halfway on one axis; complete the reveal; support it across the torso; perform one short charging-handle check with one silver glint; settle into an upward-away hero hold; hold while the glint fades.

A second masked precision edit removed only an erroneously deployed bipod from the fourth cell while preserving its barrel, sight, muzzle, actor, pose, and every other cell. The prompt prohibited firing, muzzle flash, bullets, cartridges, ammunition belts, targets, impacts, smoke, recoil, aiming at the viewer, duplicate props or limbs, random glyphs, text, logos, watermarks, and any variation in the flat `#00ff00` chroma-key background.

The initial CLI render is `output/imagegen/full-auto-v2-atlas-chromakey.png`; the corrected final is `output/imagegen/full-auto-v2-atlas-chromakey-fixed.png`, copied into the project as `design_assets/full-auto-v2-atlas-chromakey.png`. The alpha-extracted source is `design_assets/full-auto-v2-atlas-transparent.png`.
